# SQANTI3 basic statistics workflow

## Overview

This document describes the complete SQANTI3 basic statistics workflow. This module is separate from the earlier transcript recovery benchmark and the novelty/plausibility benchmark. Instead, it is a dedicated statistics and plotting workflow associated with Figure 2.

The purpose of the workflow is to generate summary tables and plot-ready inputs from SQANTI3 `classification.txt` files for the following main figure blocks:

1. **Isoform yield and SQANTI3 class composition**
2. **Isoform length distributions**
3. **Coding / non-coding composition**
4. **Number of isoforms per gene**
5. **Master figure**, which combines the four blocks into one composite figure

The workflow consists of three Python preprocessing scripts and five R plotting scripts.

## Main file names

### Python scripts

- `build_sqanti_basic_stat_table.py`
- `extract_sqanti_length_and_category_summaries.py`
- `summarize_isoforms_per_gene.py`

### R scripts

- `plot_sqanti_yield_and_category_composition.R`
- `plot_sqanti_isoform_length_distributions.R`
- `plot_sqanti_coding_noncoding_composition.R`
- `plot_sqanti_isoforms_per_gene.R`
- `assemble_sqanti_basic_stat_master_figure.R`

  Database / manifest TSV files used as input by the Python scripts.

  Final plot-ready input tables used by the R plotting scripts:

- **`category_and_length_outputs/`**

  Outputs generated by `summarize_isoforms_per_gene.py`:
  - `isoformsgene.tsv`
  - `sqanti_category_stats.tsv`

### Relationship to the `SQANTI3/` data

This workflow uses the same raw SQANTI3 `classification.txt` files that are already archived under the main `SQANTI3/` data tree by annotator × chemistry × cell line. Therefore, this module does not need to duplicate the classification files on Figshare. It is sufficient to archive the manifests and the derived summary tables separately.

## Workflow summary

```text
SQANTI3 classification.txt files + database/manifest TSV
    -> build_sqanti_basic_stat_table.py
    -> SQANTI3_basic_stat.tsv

    -> extract_sqanti_length_and_category_summaries.py
    -> sqanti_category_summary.tsv

SQANTI3 classification.txt files + database/manifest TSV
    -> summarize_isoforms_per_gene.py

    -> plot_sqanti_yield_and_category_composition.R

    -> IsoLen_MAIN
```

## `build_sqanti_basic_stat_table.py`

### What does it do?

This script reads SQANTI3 `classification.txt` files for each annotator, chemistry, and cell line from a database/manifest TSV and generates a simple summary table at the chemistry × cell × structural category × annotator level.

Among other quantities, the script summarizes:
- coding rate,
- CAGE peak support,
- polyA-site support,
- polyA motif presence,
- combined polyA-site + motif rate,
- and the number of isoforms.

This script provides the `SQANTI3_basic_stat.tsv` table used by the A and C plotting blocks.

### Example run command

```bash
python scripts/build_sqanti_basic_stat_table.py \
  -i manifests/database_with_TES.txt \
  -o Figure_input/SQANTI3_basic_stat.tsv
```

### What input does it require?

A database/manifest TSV. The minimally required columns are:

```text
Annotator    Chemical    Cell    Classification
```

The script can also handle the following optional columns:

```text
GTF    TSS    TES
```

### Example manifest

```text
Annotator    Chemical     Cell     GTF      Classification                                           TSS    TES
LoRTIA       ONT-dRNA     H1-hES   ...      /SQANTI3/LoRTIA/ONT-dRNA/H1-hES/classification.txt      ...    ...
IsoQuant     ONT-dRNA     H1-hES   ...      /SQANTI3/IsoQuant/ONT-dRNA/H1-hES/classification.txt    ...    ...
FLAIR        PacBio       WTC11    ...      /SQANTI3/FLAIR/PacBio/WTC11/classification.txt           ...    ...
```

### Which columns are required from the classification input?

The script primarily uses:

- `structural_category`
- `within_CAGE_peak`
- `within_polyA_site`
- `polyA_motif`
- `polyA_motif_found`
- `coding`

### What is the output?

The script writes a TSV table, recommended name:

- `SQANTI3_basic_stat.tsv`

The table contains at least the following columns:

```text
chemistry    cell    category    annotator    coding_rate    within_cage_rate    polyA_site_rate    polyA_motif_rate    polyA_site_and_motif_rate    n_isoforms
```

### What information does this file contain?

This file summarizes, by annotator, chemistry, cell line, and SQANTI3 category:
- isoform counts,
- coding / non-coding-related rates,
- and basic endpoint-support-related metrics.

It is the main input for:
- `plot_sqanti_yield_and_category_composition.R`
- `plot_sqanti_coding_noncoding_composition.R`

## `extract_sqanti_length_and_category_summaries.py`

### What does it do?

This script recursively traverses a root folder, identifies SQANTI3 classification-like files, and generates two types of summary outputs:

1. category summaries,
2. boxplot statistics for isoform length distributions.

The script does not operate from a manifest, but directly from the folder tree.

### Example run command

```bash
python scripts/extract_sqanti_length_and_category_summaries.py \
  -i /path/to/SQANTI3/ \
  -o category_and_length_outputs/
```

### What input does it require?

A root folder under which SQANTI3 classification-like files can be found recursively.

The files are identified from their header. The minimally required columns are:

```text
structural_category    length    coding    polyA_motif_found
```

### What is the output?

The script writes two output tables:

- `sqanti_category_summary.tsv`
- `sqanti_length_boxplot_stats.tsv`

### What information do these files contain?

#### `sqanti_category_summary.tsv`
Category-level summaries per file, including:
- counts,
- mean length,
- coding / non-coding counts and ratios,
- polyA-related ratios.

#### `sqanti_length_boxplot_stats.tsv`
Boxplot statistics for isoform length distributions per file and category:
- `min`
- `q1`
- `median`
- `q3`
- `max`
- `mean`
- `n`

### Which R script uses it?

`sqanti_length_boxplot_stats.tsv` is the direct input of:

- `plot_sqanti_isoform_length_distributions.R`

------------------------------------------

## `summarize_isoforms_per_gene.py`

### What does it do?

This script reads SQANTI3 `classification.txt` files from a database/manifest TSV and computes two types of summaries:

1. **gene-level isoform complexity**
2. **SQANTI3 structural category statistics**

This is the script that provides the gene-level isoform-count distribution used in Figure 2 block D.

### Example run command

```bash
python scripts/summarize_isoforms_per_gene.py \
  -d manifests/database_with_TES.txt \
  -o isoforms_per_gene_outputs/isoformsgene.tsv \
  --category-output isoforms_per_gene_outputs/sqanti_category_stats.tsv
```

### What input does it require?

A database/manifest TSV. Required columns:

```text
Annotator    Chemical    Cell    Classification
```

### Example manifest

```text
Annotator    Chemical     Cell     Classification
LoRTIA       ONT-dRNA     H1-hES   /SQANTI3/LoRTIA/ONT-dRNA/H1-hES/classification.txt
IsoQuant     ONT-dRNA     H1-hES   /SQANTI3/IsoQuant/ONT-dRNA/H1-hES/classification.txt
FLAIR        PacBio       WTC11    /SQANTI3/FLAIR/PacBio/WTC11/classification.txt
```

### Which columns are required from the classification input?

The script uses:

- `isoform`
- `associated_gene`
- `structural_category`

### What is the output?

The script writes two TSV files:

- `isoformsgene.tsv`
- `sqanti_category_stats.tsv`

### What information do these files contain?

#### `isoformsgene.tsv`
Gene-level isoform complexity statistics by annotator × chemistry × cell, including:

- `n_genes_total`
- `isoform_bin`
- `n_genes_in_bin`
- `frac_genes_in_bin`
- `mean_isoforms_per_gene`
- `median_isoforms_per_gene`

The `isoform_bin` bins are:

- `1`
- `2-3`
- `4-5`
- `>=6`

#### `sqanti_category_stats.tsv`
SQANTI3 structural category summaries by annotator × chemistry × cell, including:

- `n_isoforms_total`
- `structural_category`
- `n_isoforms_in_cat`
- `frac_isoforms_in_cat`

### Which R script uses it?

`isoformsgene.tsv` is the direct input of:

- `plot_sqanti_isoforms_per_gene.R`

`sqanti_category_stats.tsv` is not a mandatory input of the current Figure 2 master workflow, but it is a useful supplementary summary.

## Plot-ready inputs

The Python preprocessing scripts generate three main plot-ready input branches:

- `SQANTI3_basic_stat.tsv`
- `sqanti_length_boxplot_stats.tsv`
- `isoformsgene.tsv`

These three tables provide the inputs for the four major Figure 2 blocks.

## `plot_sqanti_yield_and_category_composition.R`

### What does it do?

This R script generates **Figure 2 block A**. It consists of two vertically stacked panels:

1. **top panel:** isoform yield by chemistry
2. **bottom panel:** SQANTI3 category composition as a stacked bar plot

The script creates the `Ablock_MAIN` object, which can be used by `assemble_sqanti_basic_stat_master_figure.R`.

### Example run command

```bash
Rscript figure_codes/plot_sqanti_yield_and_category_composition.R
```

### What input does it require?

The script reads:

- `SQANTI3_basic_stat.tsv`

### What does the input look like?

Minimally required columns:

```text
chemistry    cell    category    annotator    n_isoforms
```

### What is the output?

The script can save two main views:

- `Ablock_MAIN_cellsAVG_<method>.pdf`
- `Ablock_DETAILED_cells_3rows.pdf`

Main object used by the master figure:

- `Ablock_MAIN`

### What does it show and what is its message?

The figure shows:
- how many isoforms each annotator reconstructs across chemistries,
- and how these reconstructions are distributed across SQANTI3 categories.

It therefore summarizes both **yield** and **class composition**.

------------------------------------------

## `plot_sqanti_isoform_length_distributions.R`

### What does it do?

This R script generates **Figure 2 block B**. It plots isoform length distributions as boxplots by structural category, annotator, and chemistry.

The script exports the main panel object as `IsoLen_MAIN`.

### Example run command

```bash
Rscript figure_codes/plot_sqanti_isoform_length_distributions.R
```

### What input does it require?

The script reads:

- `sqanti_length_boxplot_stats.tsv`

### What does the input look like?

Minimally required columns:

```text
file    structural_category    min    q1    median    q3    max
```

### What is the output?

The script can write several PDFs. The main outputs include:

- `B_isoLen_<CATEGORY>_cellAVG_panel_max_cap20000.pdf`
- `B_isoLen_<CATEGORY>_cellXchem_panel_max_cap20000.pdf`
- `B_isoLen_<CATEGORY>_cellsOnly_sideBySide_panel_max_cap20000.pdf`
- `B_isoLen_MULTI_byChem_....pdf`
- `B_isoLen_MULTI_chemALL_....pdf`

Main object used by the master figure:

- `IsoLen_MAIN`

### What does it show and what is its message?

The figure shows how the length distribution of reconstructed transcript models varies:
- across SQANTI3 structural categories,
- across chemistries,
- and across annotators.

It helps assess whether different methods reconstruct systematically shorter or longer isoforms in specific classes.

------------------------------------------

## `plot_sqanti_coding_noncoding_composition.R`

### What does it do?

This R script generates **Figure 2 block C**. It displays the coding vs non-coding composition within the main SQANTI3 categories as stacked bar plots.

The script exports the main panel object as `Coding_MAIN`.

### Example run command

```bash
Rscript figure_codes/plot_sqanti_coding_noncoding_composition.R
```

### What input does it require?

The script reads:

- `SQANTI3_basic_stat.tsv`

### What does the input look like?

Minimally required columns:

```text
chemistry    cell    category    annotator    coding_rate    n_isoforms
```

### What is the output?

The script can save several views:

- `Cstyle_by_category_<method>_header.pdf`
- `Cstyle_by_category_cells_3rows_header.pdf`
- `Cstyle_cellsAVG_<method>_header.pdf`

Main object used by the master figure:

- `Coding_MAIN`

### What does it show and what is its message?

The figure shows, within FSM / ISM / NIC / NNC categories, what fraction of reconstructed isoforms are coding vs non-coding for each annotator.

This helps compare whether different annotators are biased toward more coding-like or more non-coding-like output composition.

------------------------------------------

## `plot_sqanti_isoforms_per_gene.R`

### What does it do?

This R script generates **Figure 2 block D**. It plots the distribution of the number of isoforms per gene as stacked bars using binned categories.

The script exports the main panel object as `IsoformsPerGene_MAIN`.

### Example run command

```bash
Rscript figure_codes/plot_sqanti_isoforms_per_gene.R
```

### What input does it require?

The script reads:

- `isoformsgene.tsv`

### What does the input look like?

Minimally required columns:

```text
Annotator    Chemical    Cell    n_genes_total    isoform_bin    n_genes_in_bin    frac_genes_in_bin    mean_isoforms_per_gene    median_isoforms_per_gene
```

### What is the output?

The script can write the following main PDFs:

- `IsoformsPerGene_cellsAVG_<method>_header.pdf`
- `IsoformsPerGene_cells_3rows_header.pdf`

Main object used by the master figure:

- `IsoformsPerGene_MAIN`

### What does it show and what is its message?

The figure shows, for each annotator and chemistry, what fraction of genes have:
- 1 isoform,
- 2–3 isoforms,
- 4–5 isoforms,
- or at least 6 isoforms.

It summarizes gene-level reconstruction **complexity**.

------------------------------------------

## `assemble_sqanti_basic_stat_master_figure.R`

### What does it do?

This R script generates the final composite Figure 2 master figure.

The script does not read TSV files directly. Instead, it checks whether the following four objects already exist in the current R session:

- `Ablock_MAIN`
- `IsoLen_MAIN`
- `Coding_MAIN`
- `IsoformsPerGene_MAIN`

These objects are created by the four preceding R scripts.

### Example run command

The script can only be run after the four objects have been created in the same R session. For example:

```r
source("figure_codes/plot_sqanti_yield_and_category_composition.R")
source("figure_codes/plot_sqanti_isoform_length_distributions.R")
source("figure_codes/plot_sqanti_coding_noncoding_composition.R")
source("figure_codes/plot_sqanti_isoforms_per_gene.R")
source("figure_codes/assemble_sqanti_basic_stat_master_figure.R")
```

### What is the output?

The script writes:

- `Figure2_MASTER/Figure2_ABCD_MASTER.pdf`

### What does it show and what is its message?

The master figure combines four blocks:

- **A:** isoform yield + SQANTI3 class composition
- **B:** isoform length distributions
- **C:** coding / non-coding composition
- **D:** number of isoforms per gene

It provides a complete Figure 2 summary of:
- how many isoforms the annotators reconstruct,
- which SQANTI3 classes they produce,
- how long those reconstructed transcript models are,
- what coding / non-coding composition they have,
- and how much gene-level isoform complexity they recover.

## Short summary

In this SQANTI3 basic statistics workflow:

- `build_sqanti_basic_stat_table.py` generates `SQANTI3_basic_stat.tsv`
- `extract_sqanti_length_and_category_summaries.py` generates `sqanti_length_boxplot_stats.tsv` and `sqanti_category_summary.tsv`
- `summarize_isoforms_per_gene.py` generates `isoformsgene.tsv` and `sqanti_category_stats.tsv`
- `plot_sqanti_yield_and_category_composition.R` creates block A
- `plot_sqanti_isoform_length_distributions.R` creates block B
- `plot_sqanti_coding_noncoding_composition.R` creates block C
- `plot_sqanti_isoforms_per_gene.R` creates block D
- `assemble_sqanti_basic_stat_master_figure.R` combines the four blocks into the final Figure 2 composite.
