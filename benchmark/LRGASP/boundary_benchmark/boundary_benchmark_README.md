# Boundary benchmark

## Overview

This document describes the complete boundary benchmark workflow. The aim of the pipeline is to compare transcription start site (TSS) and transcription end site (TES) detection across different annotators, sequencing chemistries, and cell lines.

The workflow handles two types of annotator input:

- For **LoRTIA**, the TSS- and TES-specific GFF3 outputs generated during the run are first converted to GTF format.
- For **the other annotators**, the input files used for the benchmark are **SQANTI3-based transcript-level GTF** files.

The boundary benchmark consists of three major parts:

1. **Boundary input generation**
   - LoRTIA TSS/TES GFF3 → GTF
   - transcript-level GTF → point-based `.TSS.gtf` and `.TES.gtf`

2. **Benchmark reference file generation**
   - global TSS/TES reference BED
   - chemistry × cell line-specific `cell_active` and `non-active_cell` reference BED files

3. **Benchmarking and statistics**
   - TP / FP / FN / precision / recall / F1 calculation
   - summary statistical tables
   - figure input tables and figure generation

## Main file names

### Python scripts

- `lortia_gff3_to_gtf.py`
- `extract_tss_tes_from_gtf.py`
- `build_sample_active_reference_beds.py`
- `benchmark_boundary_detection.py`
- `summarize_boundary_benchmark_stats.py`

### R scripts

- `plot_boundary_f1_boxpanels_per_chemistry.R`
- `plot_boundary_f1_pairwise_heatmaps_summary.R`
- `plot_boundary_f1_tss_vs_tes_scatter.R`

## Figshare structure

The name of the Figshare collection is:

`boundary_benchmark/`

The Figshare collection contains the following subdirectories:

```text
boundary_benchmark/
├── Figure_input/
│   ├── TSS_F1_score.tsv
│   ├── TES_F1_score.tsv
│   ├── LoRTIA_vs_other_pairwaise_TSS.tsv
│   └── LoRTIA_vs_other_pairwaise_TES.tsv
├── benchmark_input/
│   ├── tsspeak.bed
│   ├── hg38.PAS.main.bed
│   ├── *.TSS.gtf
│   └── *.TES.gtf
└── benchmark_reference_output/
    ├── *-cell_active.bed
    └── *-non-active_cell.bed
```

### Meaning of the subdirectories

- **`Figure_input/`**  
  Input tables used for R-based figure generation.

- **`benchmark_input/`**  
  The global reference BED files and the point-based boundary GTF files used for benchmarking:
  - `tsspeak.bed`
  - `hg38.PAS.main.bed`
  - the extracted `.TSS.gtf` and `.TES.gtf` files

- **`benchmark_reference_output/`**  
  The chemistry × cell line-specific reference files generated by the `build_sample_active_reference_beds.py` script:
  - `*-cell_active.bed`
  - `*-non-active_cell.bed`

## Workflow summary

```text
LoRTIA TSS/TES GFF3
    -> lortia_gff3_to_gtf.py
LoRTIA GTF + SQANTI3 transcript-level GTF
    -> extract_tss_tes_from_gtf.py
Point-based TSS/TES GTF + global reference BED
    -> build_sample_active_reference_beds.py
Predicted TSS/TES GTF + cell_active reference BED
    -> benchmark_boundary_detection.py
Benchmark TSV
    -> summarize_boundary_benchmark_stats.py
Figure input TSV
    -> R plotting scripts
```

## `lortia_gff3_to_gtf.py`

### What does it do?

This script converts the TSS- and TES-specific GFF3 files generated by LoRTIA into GTF format.  
In the boundary benchmark, this step is **required only for LoRTIA**.

The script can be used on:

- a single `.gff3` or `.gff3.gz` file,
- or an entire directory recursively.

### Example run command

```bash
python scripts/lortia_gff3_to_gtf.py \
  -i input/lortia_gff3/ \
  -o output/lortia_gtf/ \
  -v
```

### What input does it require?

- `.gff3` or `.gff3.gz` files generated by LoRTIA
- these may be TSS- or TES-specific boundary outputs

### What does the input look like?

The script expects GFF3 format. For the boundary benchmark, the relevant records are LoRTIA `tss` or `tes` features.

Example:

```text
chr1    LoRTIA    tss    12345    12345    .    +    .    ID=tss_001
chr1    LoRTIA    tes    12890    12890    .    +    .    ID=tes_001
```

### What is the output?

- one `.gtf` file is generated from each input GFF3 file
- these GTF files are then passed to the `extract_tss_tes_from_gtf.py` step

## `extract_tss_tes_from_gtf.py`

### What does it do?

This script generates point-based TSS and TES GTF files from transcript-level GTF files.

In the boundary benchmark, this step works from two types of input:

- for LoRTIA: the GTF files generated in the previous step
- for the other annotators: the **SQANTI3-based transcript-level GTF** files used for benchmarking

For each input GTF, the script writes two new files:

- `<sample>.TSS.gtf`
- `<sample>.TES.gtf`

### Example run command

```bash
python scripts/extract_tss_tes_from_gtf.py \
  -i input/transcript_gtf/ \
  -o output/point_gtf/ \
  -v
```

### What input does it require?

- transcript-level `.gtf` or `.gtf.gz` files
- the script uses only `transcript` feature rows

### What does the input look like?

Example transcript-level GTF row:

```text
chr1    source    transcript    1000    2000    .    +    .    gene_id "GENE1"; transcript_id "TX1";
```

From this, the script calculates the coordinates in a strand-specific way:

- TSS: `start` on the `+` strand, `end` on the `-` strand
- TES: `end` on the `+` strand, `start` on the `-` strand

### What is the output?

Two point-based GTF files:

- `*.TSS.gtf`
- `*.TES.gtf`

Example TSS output:

```text
chr1    source    transcript    1000    1000    .    +    .    gene_id "GENE1"; transcript_id "TX1"; tag "TSS";
```

Example TES output:

```text
chr1    source    transcript    2000    2000    .    +    .    gene_id "GENE1"; transcript_id "TX1"; tag "TES";
```

These files are available in the Figshare `boundary_benchmark/benchmark_input/` directory.

## `build_sample_active_reference_beds.py`

### What does it do?

This script builds chemistry × cell line-specific benchmark reference BED files.

The script:

- reads point-based `.TSS.gtf` or `.TES.gtf` files from a 3-column manifest,
- compares them to a global reference BED file,
- and writes two reference files for each chemistry × cell line combination:
  - `*-cell_active.bed`
  - `*-non-active_cell.bed`

### Example run command for TSS mode

```bash
python scripts/build_sample_active_reference_beds.py \
  -m manifests/manifest_TSS.tsv \
  -r boundary_benchmark/benchmark_input/tsspeak.bed \
  -o output/refTSS_build/ \
  --mode TSS \
  --window 10 \
  --ref-coord bed0 \
  -v
```

### Example run command for TES mode

```bash
python scripts/build_sample_active_reference_beds.py \
  -m manifests/manifest_TES.tsv \
  -r boundary_benchmark/benchmark_input/hg38.PAS.main.bed \
  -o output/refTES_build/ \
  --mode TES \
  --window 25 \
  --ref-point-mode as_is \
  -v
```

### What input does it require?

1. A **3-column manifest file**
2. A **global reference BED**
3. `--mode TSS` or `--mode TES`

### What does the input manifest look like?

The manifest contains three columns:

```text
chem    cell    gtf_path
```

Example:

```text
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-LoRTIA_annotation.TSS.gtf
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-IsoQuant_annotation.TSS.gtf
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-bambu_annotation.TSS.gtf
```

### What does the TSS reference BED look like?

The TSS reference is a BED6 peak/cluster file:

```text
chr1    16013    16020    rfhg_1.1    1    -
chr1    29346    29366    rfhg_2.1    1    -
chr1    36521    36538    rfhg_3.1    1    +
```

### What does the TES reference BED look like?

The TES reference is a BED6 point file:

```text
chr1    940182    940182    SAMD11    1    +
chr1    944292    944292    SAMD11    1    +
chr1    932116    932116    SAMD11    1    +
```

The `tsspeak.bed` and `hg38.PAS.main.bed` files are available in the Figshare `boundary_benchmark/benchmark_input/` directory.

### What is the output?

The script writes chemistry × cell line-specific reference files:

- `<chem>-<cell>-cell_active.bed`
- `<chem>-<cell>-non-active_cell.bed`

These generated benchmark reference files are available in the Figshare `boundary_benchmark/benchmark_reference_output/` directory.

## `benchmark_boundary_detection.py`

### What does it do?

This script runs the actual boundary benchmark.

The script:

- reads the point-based TSS/TES GTF files of the annotators,
- reads the chemistry × cell line-specific `cell_active` reference BED files,
- and calculates the boundary detection metrics.

The output metrics include:

- `tp`
- `fp`
- `fn`
- `precision`
- `recall`
- `f1`
- `n_pred`
- `n_ref`

### Example run command for TSS mode

```bash
python scripts/benchmark_boundary_detection.py \
  -m manifests/manifest_reference_TSS.tsv \
  --mode TSS \
  --window 10 \
  --ref-coord bed0 \
  --chr-mode keep \
  -o results/TSS_metrics.tsv \
  -v
```

### Example run command for TES mode

```bash
python scripts/benchmark_boundary_detection.py \
  -m manifests/manifest_reference_TES.tsv \
  --mode TES \
  --window 25 \
  --ref-point-mode as_is \
  --chr-mode keep \
  -o results/TES_metrics.tsv \
  -v
```

### What input does it require?

The script expects a **4-column manifest**.

### What does the input manifest look like?

The four columns are:

```text
Chemistry    Cell-line    GTF    Reference
```

Example:

```text
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-LoRTIA_annotation.TSS.gtf    /path/to/ONT-CapTrap-H1-cell_active.bed
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-IsoQuant_annotation.TSS.gtf  /path/to/ONT-CapTrap-H1-cell_active.bed
ONT-CapTrap    H1    /path/to/H1-ONT-CapTrap-bambu_annotation.TSS.gtf     /path/to/ONT-CapTrap-H1-cell_active.bed
```

### What is the output?

The script writes a TSV table. The main columns are:

```text
mode    window    chem    cell    annotator    tp    fp    fn    precision    recall    f1    n_pred    n_ref    gtf    ref
```

These benchmark TSV files provide the input for the next statistical summarization step.

## `summarize_boundary_benchmark_stats.py`

### What does it do?

This script generates summary statistical tables from the benchmark TSV files.

The script:

- writes a mean summary table,
- creates an ANOVA table,
- writes the LoRTIA vs. other annotators comparison,
- and can optionally generate a cell line pair comparison table.

### Example run command

```bash
python scripts/summarize_boundary_benchmark_stats.py \
  -i results/TSS_metrics.tsv results/TES_metrics.tsv \
  -o results/boundary_stats \
  --lortia-name LoRTIA \
  --padj bh
```

### What input does it require?

The script expects one or more benchmark TSV files generated by `benchmark_boundary_detection.py`.

### What does the input look like?

The minimally required columns are:

```text
mode    window    chem    cell    annotator    f1
```

In practice, the script uses the full benchmark output.

### What is the output?

Using the specified prefix, the script writes the following files:

- `boundary_stats_means.tsv`
- `boundary_stats_anova.tsv`
- `boundary_stats_lortia_vs_others.tsv`

If optional cell line comparisons are also requested, then:

- `boundary_stats_cellline_pairs.tsv`

## `plot_boundary_f1_boxpanels_per_chemistry.R`

### What does it do?

This R script generates a chemistry-faceted TSS and TES F1 boxpanel figure.

The script:

- reads the TSS and TES F1 score tables,
- draws annotator-wise boxplots,
- overlays cell line-specific points next to the boxes,
- separates the figure into chemistry-specific panels,
- and saves the result as a PDF.

### Example run command

```bash
Rscript scripts/plot_boundary_f1_boxpanels_per_chemistry.R
```

### What input does it require?

From the Figshare `boundary_benchmark/Figure_input/` directory:

- `TSS_F1_score.tsv`
- `TES_F1_score.tsv`

### What does the input look like?

The score tables must contain the following columns:

```text
Chemistry    Annotator    Cell line    F1
```

### What is the output?

The script produces:

- `PCR_TSS_TES_F1_boxpanel_perChem_minmax.pdf`

### What does it do?

This R script generates a multi-panel summary figure.

The figure contains:

- TSS F1 boxpanel
- TES F1 boxpanel
- LoRTIA vs. other annotators TSS pairwise heatmap
- LoRTIA vs. other annotators TES pairwise heatmap
- TSS vs TES F1 scatter panel

### Example run command

```bash
Rscript scripts/plot_boundary_f1_pairwise_heatmaps_summary.R
```

### What input does it require?

From the Figshare `boundary_benchmark/Figure_input/` directory:

- `TSS_F1_score.tsv`
- `TES_F1_score.tsv`
- `LoRTIA_vs_other_pairwaise_TSS.tsv`
- `LoRTIA_vs_other_pairwaise_TES.tsv`

### What does the input look like?

#### Required columns in the score tables

```text
Chemistry    Annotator    Cell line    F1
```

#### Required columns in the pairwise tables

```text
Chemistry    Annotator_1    Annotator_2    mean_diff_F1_(a1_minus_a2)    p_adj_BH
```

The script produces:

- `F1_master_figure.pdf`

## `plot_boundary_f1_tss_vs_tes_scatter.R`

### What does it do?

This R script generates only the scatter plot between TSS and TES F1 values.

The script:

- reads the TSS and TES F1 score tables,
- matches them using the `Cell line + Chemistry + Annotator` key,
- places `F1_TSS` on the x-axis and `F1_TES` on the y-axis,
- uses point colour to represent annotator,
- and uses point shape to represent chemistry.

### Example run command

```bash
Rscript scripts/plot_boundary_f1_tss_vs_tes_scatter.R
```

### What input does it require?

From the Figshare `boundary_benchmark/Figure_input/` directory:

- `TSS_F1_score.tsv`
- `TES_F1_score.tsv`

### What does the input look like?

The two score tables must contain the following columns:

```text
Chemistry    Annotator    Cell line    F1
```

The script joins the two tables using the following keys:

- `Cell line`
- `Chemistry`
- `Annotator`

### What is the output?

The script produces:

- `TSS_vs_TES_scatter.pdf`

## Short summary

In the boundary benchmark:

- the **global benchmark input files** are stored in the Figshare `boundary_benchmark/benchmark_input/` directory,
- the **generated chemistry × cell line-specific reference files** are stored in the Figshare `boundary_benchmark/benchmark_reference_output/` directory,
- the **figure input tables** are stored in the Figshare `boundary_benchmark/Figure_input/` directory,
- the benchmarking workflow consists of the five Python scripts listed above,
- and the figure generation workflow consists of the three R scripts listed above.
